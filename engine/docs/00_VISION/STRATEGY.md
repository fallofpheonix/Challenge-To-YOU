# Project Chrysalis: Strategy & Vision

## The Core Loop
The Architect's Swarm is a game about **systems engineering and debugging**. The player never directly controls a drone; they write the logic that governs the swarm and observe the emergent outcomes.

## Narrative Arc: The Evolution of the Architect

| Phase | Identity | Primary Verb | The Adversary |
| :--- | :--- | :--- | :--- |
| **Act I: The Drop** | **Teacher** | Write basic logic, form supply chains | Entropy & Initial Bugs |
| **Act II: The Ecosystem** | **Engineer** | Patch logic for resilience, sacrifice patterns | Deterministic Hazards |
| **Act III: The Ghost** | **Detective** | Diagnose drift, manage Byzantine faults | Competing Intelligence |
| **Epilogue: The Silent Act**| **Witness** | Observe unplanned emergence | None (The System itself) |

## Design Pillars

1. **Systems over Content:** Focus on deep, emergent interactions between simple rules rather than handcrafted levels.
2. **Visual Spectacle of Cause and Effect:** Failure must be loud, physical, and entertaining (e.g., drone pile-ups, circular marching) rather than silent crashes.
3. **Diegetic Telemetry:** All UI exists as orbital telemetry data. Use heatmaps and density fluids instead of 10,000 individual unit icons.
4. **Resilience as Victory:** Winning is defined by the system's ability to self-heal and maintain baseline survival under extreme stress, not by "destroying" an enemy.

## Key Mechanics

- **Digital Pheromones:** Localized mesh signals drones leave behind to guide others.
- **Byzantine Fault Tolerance:** Late-game mechanics requiring trust-scoring and quorum consensus to handle compromised drones.
- **Uplink Windows:** Limited bandwidth creates tension—you must trust your code to run autonomously between communication bursts.
- **Logic Weaponization:** The "Logic Bomb" of Act III is not a new weapon, but a clever repurposing of the existing supply chain infrastructure.
