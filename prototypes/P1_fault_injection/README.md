# P1: Fault Injection

## Goal

Answer one question:

> Can a single controllable perturbation create enough curiosity that players voluntarily repeat experiments?

## Core Question

Can one tiny intervention create enough curiosity that players voluntarily repeat experiments?

## Maturity

P1 — Interactive prototype

## Scope

The scene contains only:

- Sensor
- Door
- Timer
- Alarm

The player performs exactly one action:

- Inject delay

## Rules

- One interaction
- One puzzle
- No AI
- No progression
- No story
- No audio
- No save system
- No replay system
- No polish
- No scripting unless the prototype specifically tests scripting
- No roguelike generation
- No inventory
- No upgrades
- No multiple puzzles

## Success Criteria

- Players voluntarily repeat the interaction
- Players ask "what happens if..." without prompting
- Players can explain the result afterward
- Players tell at least one story about the outcome
- Players choose a non-obvious action at least once
- Players explicitly ask to try another variation

## Failure Criteria

- Players perform only the obvious action
- Players get the answer and stop experimenting
- Players cannot explain why the outcome happened
- Players never produce a hypothesis unprompted
