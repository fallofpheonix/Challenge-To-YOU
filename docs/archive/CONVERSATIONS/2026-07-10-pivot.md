## Date: 2026-07-10
## Participants: User + AI Assistant
## Topic: Major Project Scope Change

## Summary

User announced major changes and scope change for Project Chrysalis. Through iterative questioning, we determined the new direction is a **completely different project** — a gamified coding challenge platform called **"Challenge To YOU"**.

---

## Key Decisions Made

### 1. Project Identity
- **Old Name**: Project Chrysalis: The Architect's Swarm
- **New Name**: Challenge To YOU
- **Type**: Roguelike coding puzzle game
- **Platform**: Desktop (Godot 4)

### 2. Core Concept
A multi-era hacking game where players solve procedurally generated coding challenges across fantasy/sci-fi themes. The core mechanic is **Emergent Multi-Layer Systems** — combining broken/unrelated code to create glitches, loopholes, and side-effects that produce passcodes.

### 3. Era Progression (v1 = 2 Eras)
| Era | Theme | Code Type |
|-----|-------|-----------|
| Tier 1 | Medieval Magitech | Custom DSL (Runes) |
| Tier 2 | Cyberpunk Neon | Real scripting (Python/JS) |

### 4. Gameplay Modes (v1 = 3 Modes)
| Mode | Role | Objective |
|------|------|-----------|
| Architect | Builder | Write clean new modules |
| Ghost | Stealth Hacker | Modify code under AI detection |
| Saboteur | Chaos Agent | Break code for chain reactions |

### 5. Core Mechanics
1. **Procedural Generation**: Seed-based RNG stitches modular code segments
2. **Luck Mechanic**: Affects level complexity and AI monitoring
3. **Multi-Layer Code**: Frankenstein code, loophole exploitation
4. **Dynamic Passcodes**: Emerge from code interactions, not direct output

### 6. Tech Stack
| Layer | Technology |
|-------|------------|
| Frontend | Godot 4 (GDScript/C#) |
| Backend | Go 1.26+ |
| Sandbox | WASM (Extism/Wasmer) |
| AI | Ollama + Llama 3 |

### 7. Launch Strategy
- **Week 4**: Free/PWYW alpha on Itch.io
- **Month 3**: Steam Early Access ($4.99-$9.99)
- **Month 6**: Full Steam release

### 8. Scope for MVP
- 2 eras (Magitech → Cyberpunk)
- 3 modes (Architect, Ghost, Saboteur)
- 15-20 modular code segments
- Procedural generation with luck mechanic
- Skip WASM for alpha (use text parsing)
