# Educational Mapping & Curriculum Alignment

This document aligns the **Challenge To YOU** puzzles with Bloom's Taxonomy of Cognitive Learning and the ACM/IEEE Computer Science Curriculum guidelines.

---

## 1. Bloom's Taxonomy Mapping

Every puzzle is classified based on the target cognitive level:

### 1.1 Remember (Knowledge Retrieval)
- **Objective**: Recall basic terminology and syntax structures.
- **Puzzles**: Syntax-highlighting corrections, naming registers, listing basic SQL tables.
- **Example**: `M-01: Runic Initiation` (variables declaration).

### 1.2 Understand (Comprehension)
- **Objective**: Explain facts, trace simple executions, and outline data routes.
- **Puzzles**: Code reviews, trace audits, predicting branch targets.
- **Example**: `R-202: Merge Fix` (understanding git merge conflict tags).

### 1.3 Apply (Implementation)
- **Objective**: Write functioning scripts to solve logical and system objectives.
- **Puzzles**: Coding challenges, writing SQL queries, deploying Kubernetes pods.
- **Example**: `B-282: CRISPR Splice` (gene splicing).

### 1.4 Analyze (Diagnostic)
- **Objective**: Trace complex asynchronous systems, locate memory leaks, and debug deadlocks.
- **Puzzles**: Bug hunting, identifying data hazards, scanning packet streams.
- **Example**: `C-42: Thread Race` (tracing concurrent threads).

### 1.5 Evaluate (Optimisation)
- **Objective**: Compare alternative solutions and optimize cycle count or memory footprint.
- **Puzzles**: Refactoring scripts, reducing instruction budgets, load balancing configurations.
- **Example**: `A-402: Pipeline Schedule` (scheduling assembly).

### 1.6 Create (Synthesis)
- **Objective**: Design complete system architectures and write custom compiler parsers or operating system handlers.
- **Puzzles**: Boss fights, writing parsers, designing custom panic handlers.
- **Example**: `K-524: Panic Recover` (writing operating system ISRs).

## 2. Academic Curriculum Integration (ACM/IEEE CS Map)

Our challenges map directly to core courses in the ACM/IEEE Computer Science Curriculum:

- **CS-101 (Intro to Programming)**: Covered in *Medieval Magitech* (variables, control loops, functions).
- **CS-201 (Data Structures)**: Covered in *Cosmic Void* and *Dream Protocol* (binary trees, graphs, traversals).
- **CS-301 (Operating Systems)**: Covered in *The Kernel Beyond* (scheduling, pagination, ISRs).
- **CS-302 (Computer Architecture)**: Covered in *Machine Cathedral* (registers, pipelines, CPU cycles).
- **CS-401 (Networking)**: Covered in *Oceanic Grid* and *Cloud Dominion* (routing, consensus, scaling).
- **CS-402 (Cybersecurity)**: Covered in *Cipher Realm* and *Eclipse Dominion* (encryption, buffer overflows, injection).

## 3. Skill Unlock Tree & Concept Progression

The learning tree defines prerequisites for level gates:

```
[Syntax Basics] ──► [Conditionals] ──► [Loops] ──► [Recursion]
      │
      ▼
[Memory Allocation] ──► [Pointers] ──► [Registers] ──► [Page Tables]
```

### Milestone Targets & Expected Completion Times:
1. **Rune Binder (Beginner)**: Basic variables, loops, recursion. (Expected time: 10 Hours).
2. **System Auditor (Intermediate)**: Concurrency, simple registers, networks. (Expected time: 25 Hours).
3. **Kernel Ascendant (Master)**: Assembly pipelines, custom page tables, kernel panic ISRs. (Expected time: 60 Hours).
