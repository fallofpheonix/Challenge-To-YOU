# Master Challenge Taxonomy: Part 2 (Categories 11-20)

This document defines the final ten categories of the **Challenge To YOU** master challenge ecosystem, focusing on AI, Cloud, Compilers, and Cognitive skill loops.

---

## 11. Artificial Intelligence
- **Description**: Manipulate neural network weights, construct prompt injections, align attention parameters, and navigate token spaces.
- **Purpose**: Teach machine learning basics, transformer logic, prompt engineering, and model security.
- **Learning Objectives**: Inject prompt constraints, adjust network weights, bias token classification boundaries.
- **Difficulty Curve**: 5-10. Simple prompts up to gradient calculation and poisoning.
- **Player Fantasy**: Cognitive poisoner manipulating AI sentinels to mask their presence.
- **Required Knowledge**: Machine learning basics, vector calculations, prompt contexts.
- **Subcategories**: Prompt engineering, weight poisoning, semantic offsets, model jailbreaking.
- **Algorithms Covered**: Gradient descent optimization, semantic embedding routing.
- **Common Mistakes**: Directly stating commands that trigger simple keyword filters.
- **Evaluation Methods**: Model output validation inside the neural network simulator.
- **Replay Opportunities**: Jailbreaking sentinels using minimal prompt length.
- **Scaling Strategy**: Add active safety filters, increase parameter count.
- **Mission Integration**: Overriding AI scanners in neural mainframes.
- **Universe Mapping**: Neural Labyrinth.
- **Reward Types**: Data tokens, gradient charts, XP.

---

## 12. Databases
- **Description**: Write SQL and JSON queries, build indexes, manage concurrency locks, and optimize schemas.
- **Purpose**: Teach database theory, normalization, indexing, and transactional ACID rules.
- **Learning Objectives**: Normalize schemas, build indices, resolve deadlocks, enforce ACID properties.
- **Difficulty Curve**: 3-10. Basic SQL queries to nested transactions and index optimization.
- **Player Fantasy**: Relational explorer navigating schema temples.
- **Required Knowledge**: Schema relations, ACID rules, tree-indexing mechanics.
- **Subcategories**: SQL scripting, JSON databases, index optimization, transaction locking.
- **Algorithms Covered**: Index B-Tree traversals, ACID transaction isolation.
- **Common Mistakes**: Creating nested deadlock locks, performing full table scans.
- **Evaluation Methods**: Schema query optimizer validation measuring search operations.
- **Replay Opportunities**: Optimize queries to retrieve records in fewer steps.
- **Scaling Strategy**: Spike record counts, add concurrent connection loads.
- **Mission Integration**: Querying historical data tables to unlock door permissions.
- **Universe Mapping**: Data Abyss.
- **Reward Types**: Schema parts, query tokens, XP.

---

## 13. Cloud Computing
- **Description**: orchestrate container deployments, write service specifications (YAML), scale resources, and configure load balancers.
- **Purpose**: Teach virtualization, container scaling, load balancing, and distributed hosting structures.
- **Learning Objectives**: Write YAML deployment configs, allocate cycle/memory limits, set load balances.
- **Difficulty Curve**: 4-10. Basic specs to container replica set scaling under load spikes.
- **Player Fantasy**: Cluster architect scaling cloud citadels to process traffic.
- **Required Knowledge**: Containers, YAML specification structures, network routing rules.
- **Subcategories**: Pod configuration, cluster scaling, load balancing, service mesh design.
- **Algorithms Covered**: Least-Connections routing, replica scaling checks.
- **Common Mistakes**: Exceeding pod memory boundaries, creating dependency loops.
- **Evaluation Methods**: Cloud deployment check verifying container status under traffic loads.
- **Replay Opportunities**: Deploying clusters with minimal resource allocations.
- **Scaling Strategy**: Trigger mock DDoS packet floods, drop cluster nodes.
- **Mission Integration**: Restoring server bank services to power up transit portals.
- **Universe Mapping**: Cloud Dominion.
- **Reward Types**: Cloud codes, proxy cores, XP.

## 14. Graphics
- **Description**: Write fragment and vertex shaders (GLSL), calculate lighting models, and manipulate coordinate spaces.
- **Purpose**: Teach 3D rendering mathematics, lighting formulas (Phong/PBR), and GPU vertex transformations.
- **Learning Objectives**: Map coordinates, calculate vector dot products, write fragment color shaders.
- **Difficulty Curve**: 5-10. Simple textures to custom PBR Specular lighting shaders.
- **Player Fantasy**: Dimensional painter rendering portals to restore visual fields.
- **Required Knowledge**: Vector geometry, GLSL coding syntax, Phong/PBR equations.
- **Subcategories**: Fragment shaders, vertex offsets, ray-tracing math, projection mapping.
- **Algorithms Covered**: Blinn-Phong lighting, Ray-Sphere intersection equations.
- **Common Mistakes**: Dividing variables by zero, mapping incorrect coordinate spaces.
- **Evaluation Methods**: Rendering viewport check measuring output color and refresh rates.
- **Replay Opportunities**: Optimizing shaders to compile with fewer operations.
- **Scaling Strategy**: Increase vertex complexity, add multiple light vectors.
- **Mission Integration**: Repairing mirror portals to traverse coordinate splits.
- **Universe Mapping**: Fractal Dream.
- **Reward Types**: Shader keys, texel chips, XP.

## 15. Game Development
- **Description**: Configure state machines, write collision responses, and implement pathfinding (A*) and ECS (Entity Component System) logic.
- **Purpose**: Teach game design patterns, coordinate math, physics simulations, and state transitions.
- **Learning Objectives**: Chart state transitions, calculate path routes, configure entity components.
- **Difficulty Curve**: 3-10. Simple state gates to complex A* route navigation.
- **Player Fantasy**: Summoner coding AI behavior components for automated golems.
- **Required Knowledge**: Coordinate systems, state transitions, grid routing algorithms.
- **Subcategories**: Pathfinding logic, state machine design, collision scripting, ECS layouts.
- **Algorithms Covered**: A* pathfinding, FSM (Finite State Machine) transitions.
- **Common Mistakes**: Infinite state loops, failing to resolve collision overlaps.
- **Evaluation Methods**: Physics simulator checks tracking entity path resolution to targets.
- **Replay Opportunities**: Solving routes in fewer steps or minimal path code size.
- **Scaling Strategy**: Add dynamic obstacles, scale grid boundaries.
- **Mission Integration**: Programming automation routines for drone patrols.
- **Universe Mapping**: Medieval Magitech, Silicon Wastes.
- **Reward Types**: State cores, path guides, XP.

## 16. Compiler Engineering
- **Description**: Construct lexers and parsers, build AST structures, and optimize Intermediate Representation (IR).
- **Purpose**: Teach parser theory, formal grammar, AST optimization, and code compilation steps.
- **Learning Objectives**: Define lexer regular expressions, construct parsing grammars, balance AST trees.
- **Difficulty Curve**: 5-10. Simple regex to complete grammar parsers and compiler optimization passes.
- **Player Fantasy**: Syntactical scholar repairing the compiling libraries of knowledge.
- **Required Knowledge**: Regular expressions, BNF parsing grammars, compiler theory.
- **Subcategories**: Lexical scanners, grammar parsers, AST trees, IR optimization passes.
- **Algorithms Covered**: Recursive descent parsing, AST tree balancing.
- **Common Mistakes**: Writing ambiguous parsing grammars, creating stack overflows.
- **Evaluation Methods**: Compiler parser checks verifying syntax parsing accuracy.
- **Replay Opportunities**: Writing grammars with minimal syntax tokens.
- **Scaling Strategy**: Add dynamic types, complicate parsing token dictionaries.
- **Mission Integration**: Repairing compiler engines in central libraries.
- **Universe Mapping**: Aether Library, Cosmic Void.
- **Reward Types**: Grammar inks, parsing quills, XP.

## 17. Human Cognitive Challenges
- **Description**: Logic puzzles that tax working memory, attention switching, spatial planning, and system-level thinking.
- **Purpose**: Develop raw cognitive skills, focus, and logical planning under time limits.
- **Learning Objectives**: Retain register values in memory, switch attention between parallel loops, plan spatial moves.
- **Difficulty Curve**: 1-10. Simple matches to multi-level spatial rotations and variable tracking.
- **Player Fantasy**: Mental hacker mapping cognitive node paths in their mind.
- **Required Knowledge**: Basic spatial coordinates.
- **Common Mistakes**: Making quick edits without auditing the entire coordinate network.
- **Advanced Variations**: Solving logic paths while memory values rotate in space.
- **Boss Versions**: Cortex (cognitive maze navigation).
- **Procedural Generation Rules**: Generate dynamic logic grids and coordinate shifts.
- **Replayability Score**: 75/100.
- **Universe Mapping**: Neural Labyrinth.
- **Reward Types**: Focus drops, memory chips, XP.

- **Required Knowledge**: Time/space complexity of data structures, API protocol types.
- **Subcategories**: Spec auditing, structure selection, API design, trade-off reviews.
- **Algorithms Covered**: Complexity profiling, load analysis.
- **Common Mistakes**: Selecting slow queue components, overlooking scalability bounds.
- **Evaluation Methods**: Architecture compiler check testing data flow constraints.
- **Replay Opportunities**: Optimizing data flows to minimize operational cost.
- **Scaling Strategy**: Add data throughput limits, complicate requirement specifications.
- **Mission Integration**: Designing scheduling systems for automatic metal smelters.
- **Universe Mapping**: Forge Prime.

## 20. Research Challenges
- **Description**: Read RFC documentation, technical specs, and diagnostic logs to extract implementation details.
- **Purpose**: Develop independent research, reading comprehension, and spec-implementation skills.
- **Learning Objectives**: Read technical RFCs, identify connection parameters, locate system values.
- **Difficulty Curve**: 4-10. Simple spec searches to parsing complete network protocols.
- **Player Fantasy**: Cryptographic scholar recovering lost configuration details from raw memory dumps.
- **Required Knowledge**: RFC structures, diagnostic log formats.
- **Subcategories**: Specs analysis, log auditing, RFC checks, document parsing.
- **Algorithms Covered**: String parsing, diagnostic searches.
- **Common Mistakes**: Misinterpreting spec boundaries, using deprecated configurations.
- **Evaluation Methods**: Input checks verifying implementation details against reference spec data.
- **Replay Opportunities**: Parsing logs with minimized query sizes.
- **Scaling Strategy**: Obfuscate log archives, extend length of technical specs.
- **Mission Integration**: Reconstructing BIOS boot settings from corrupted memory sectors.
- **Universe Mapping**: Origin Kernel.
- **Reward Types**: Research logs, code indexes, XP.
